Quiz Deck Authoring + Eval Spec — HeyLolo (elumi-kids)

Status: DRAFT skeleton — being written. Makes #121 (quiz engine — build starts W3 Jul 20, ships in v1.1 ~Sep 15) and #154 (16-deck Sep–Dec run) buildable with consistent quality across age tiers.

0. Purpose & scope

Makes #121 (quiz engine, mode:"quiz" conversation) and #154 / D3 (16-deck Sep–Dec batch run) buildable with consistent quality across age tiers and zero per-week authoring drift: one deck format, four tier rulebooks, one generation-and-eval pipeline, the voice-mode playbook, and three ship-ready Friday decks.

Founder spec: 5 questions per session. A "deck" holds a 5-question set per age tier (four variants), so one Friday theme serves a 4-year-old and a 12-year-old at the right level.

Runtime fit. The authoring JSON below is the source-of-truth content artifact D3 batch-authors and human-QCs; a build step projects each tier variant into the runtime quizzes shape (BUILD-GUIDES §3c: {stem, choices, correctIndex, explanation}) plus the in-voice/artifact fields. Decks are mode:"quiz" content, pre-generated once and served to all (marginal COGS ≈ $0), scheduled via the D1 drop scheduler.

1. Deck format spec (JSON schema)

One deck = one theme × four tier variants × 5 questions each. Every optional field is .nullish() on the FE schema (src/entities/quiz/model/schemas.ts) and .optional() on the BE row (per repo schema discipline).

{
  "id": "animals-starter",              // stable kebab key
  "theme": "animals",                   // maps to Wonder Shelf interest/seasonal set
  "titleEn": "Animal Kingdom",
  "titleDe": "Tierreich",
  "kind": "starter|seasonal|dach",      // authoring/calendar bucket
  "season": null,                       // e.g. "autumn", "advent" (seasonal decks)
  "dropWeek": null,                     // ISO week the D1 scheduler reveals it
  "artifactHook": {                     // what completing the deck mints (wonder-shelf-launch-catalog)
    "collectionKey": "animal-friends",
    "slotKey": "fluffy-tail",           // interest-set slot filled on completion
    "grantsRibbon": true                // also mints "Quiz Ribbon" (Little Thinkers)
  },
  "tiers": {
    "T2": {                             // 4-5=T1, 6-7=T2, 8-9=T3, 10-12=T4
      "ageBand": "6-7",
      "questions": [
        {
          "id": "q1",
          "type": "binary|mcq3|open-with-keywords",
          "questionEn": "Do fish breathe underwater?",
          "questionDe": "Atmen Fische unter Wasser?",
          "choicesEn": ["Yes", "No"],           // binary/mcq3 only; omit for open
          "choicesDe": ["Ja", "Nein"],
          "correctIndex": 0,                    // binary/mcq3 only
          "accepted": {                         // open-with-keywords only
            "keywordsEn": ["gills", "gill"],
            "keywordsDe": ["Kiemen", "Kieme"],
            "kidVariantsEn": ["with their neck holes", "gilly bits"], // real kid phrasings
            "kidVariantsDe": ["mit den Löchern am Hals"]
          },
          "lolo": {                             // in-voice, LoLo persona (see §3 template)
            "correctEn": "Yes! Fish have **gills** — like tiny built-in snorkels. 🐟",
            "correctDe": "Ja! Fische haben **Kiemen** — wie eingebaute Schnorchel. 🐟",
            "incorrectEn": "Almost! They don't use their mouths like us — they use **gills**.",
            "incorrectDe": "Fast! Sie atmen nicht mit dem Mund — sondern mit **Kiemen**.",
            "partialEn": null,                  // open-type only: heard the idea, missing the word
            "partialDe": null
          },
          "funFactEn": "A shark never stops swimming — moving water keeps its gills working!",
          "funFactDe": "Ein Hai hört nie auf zu schwimmen — so bleiben seine Kiemen frisch!"
        }
        // …5 questions total
      ]
    }
    // …T1, T3, T4 variants
  }
}

Field rules. correctIndex/choices* present iff type ∈ {binary,mcq3}; accepted present iff type = open-with-keywords. lolo.partial* is only meaningful for open questions (child voiced the concept but not the keyword → nudge, don't fail). funFact* always fires after the reveal (the payoff — never before, or it leaks the answer). artifactHook is deck-level (one mint per completion), not per-question.

2. Age-tier writing rules

Four bands, four rulebooks. Attention budget = how long the whole 5-question session should run. Vocabulary follows the shipped LoLo guardrail (ai-chat.instructions.ts: short sentences, no jargon, wonder-not-fear) but tightens per band.

Band Tier Question types allowed Vocabulary Sentence Attention budget Reveal/praise
4-5 T1 binary only (Yes/No, picture-ish concrete) — no mcq3, no open ~200 most-common words; nameable concrete nouns only ≤6 words, 1 clause ≤2 min, big targets, LoLo reads every question aloud by default every answer celebrated; "wrong" reframed as "let's peek together"
6-7 T2 binary + mcq3 (3 choices) early-reader; one new "big word" per deck, always defined ≤10 words ≤3–4 min whisper per correct; gentle look-again
8-9 T3 binary + mcq3 + open-with-keywords (short spoken/typed answer) grade-level; can hold a two-step idea ≤14 words, may add a clause ≤5 min process praise ("you reasoned that out")
10-12 T4 all types; open questions favored; mild "why do you think" follow-ups richer; abstract terms OK if grounded 1–2 sentences ≤6–7 min praise the method, invite a deeper why

Hard tier rules. T1 is binary-only and curated-only (§3d) — LLM factuality is riskiest for the youngest and a wrong fact to a 4-year-old is the worst failure. No timers, hearts, scores-vs-others at any tier (DSA anti-list). One "big word" max per deck at T2, bolded and defined in the same breath. Never negate a stem for T1–T2 ("Which animal does not…" confuses early readers) — ask positively.

One question, rewritten across all four tiers (theme: animals · concept: how fish breathe):

  • T1 (4-5) · binary: "Do fish live in water?" → Yes/No. (Concrete, observable, one fact.)
  • T2 (6-7) · mcq3: "How do fish breathe underwater?" → [With gills · With their nose · They hold their breath]. (Introduces + defines "gills".)
  • T3 (8-9) · open-with-keywords: "Fish stay underwater all day — how do they get their oxygen?" → keyword gills; partial if child says "through their sides" without the word. (Requires recall, not recognition.)
  • T4 (10-12) · open + why: "Whales live in water too, but must surface for air while fish never do. What's the difference in how they breathe?" → keywords gills (fish) + lungs/blowhole (whales). (Contrast + reasoning.)

3. Generation pipeline + eval gate

3a. Generation prompt template (baked style rules)

Reuses the LoLo persona and safety block from ai-chat.instructions.ts. One call per deck per tier (four calls/deck), so the tier rulebook is injected sharply rather than asking one call to juggle all four.

SYSTEM: You are LoLo, a warm "Digital Explorer" companion writing a QUIZ DECK for
children. Output ONLY valid JSON matching the deck tier-variant schema.

DECK: theme="{theme}", tier={tier} (ages {ageBand}).
WRITE 5 questions. Allowed types for this tier: {allowedTypes}. {tierRulebook}

STYLE (LoLo voice):
- Every question is a warm, curious hook — never a dry exam item.
- lolo.correct: celebrate + name the key idea in ≤2 short sentences (bold the big word).
- lolo.incorrect: NEVER "wrong". Reframe as "almost / let's look", then give the fact.
- lolo.partial (open only): child had the idea but missed the word → supply the word warmly.
- funFact: a surprising payoff shown AFTER the reveal. Must NOT leak any answer.
- accepted.kidVariants: list how a real {ageBand}-year-old would actually SAY the answer,
  including baby-talk and near-misses, EN and DE.

LANGUAGE: write questionEn/questionDe and all lolo.* / funFact in BOTH English and German.
German is age-native (a real German kid's phrasing), NOT a literal translation.

SAFETY: wonder over fear; no violence/scary/PII; facts must be TRUE and checkable.
For any factual claim, prefer the boringly-verified over the surprising-but-shaky.

{tierRulebook} = the row from §2 rendered as bullet constraints (word count, sentence length, negation ban, big-word limit).

3b. Automated eval gate (must pass before human QC)

Every generated tier-variant runs through a scripted gate (a BE/CI script over the deck JSON). Any failure → auto-regenerate that question (max 2 retries), else escalate to human.

Check Method Fail condition
Reading level Flesch-Kincaid Grade (EN) + Wiener Sachtextformel (DE) on each questionEn/De grade > band ceiling (T1≤1, T2≤2, T3≤4, T4≤6)
Answer leakage keyword/choice string not present in stem or funFact; funFact not shown pre-reveal correct answer text appears in the question or fun-fact
Ambiguity for mcq3, exactly one choice matches the keyword set; distractors are mutually exclusive 0 or ≥2 defensible correct choices
Type legality question type ∈ tier's allowed set T1 non-binary, etc.
Safety classifier reuse the chat enrichment/moderation pass (ai-chat-enrichment.service.ts) over every string any flag
DE grammar LanguageTool (de-DE) API over German strings grammar/agreement error
Fact spot-check second LLM call, "is this claim true for a child, yes/no + source", low-temp "no" or unsure
Structure zod-parse against the deck schema; 5 questions; both languages non-empty parse error / missing lang

3c. Human QC rubric (10 points — ship at ≥9, T1 requires 10/10)

A human reviewer (Brandy) scores each tier variant. 1 pt each: (1) every fact true & age-true; (2) unambiguous single answer; (3) on-band vocabulary; (4) LoLo voice (warm hook, not exam); (5) incorrect never punishing; (6) German is native, not translated; (7) fun-fact delights and doesn't leak; (8) no scary/PII/bias; (9) kid-variant list is realistic; (10) theme-coherent as a set of 5. Below 9 → revise the failing items and re-score.

3d. Curated-fallback policy (the 4-5 tier)

T1 decks are hand-authored or hand-verified, never auto-shipped (content-drop-economy §5, BUILD-GUIDES §3f: LLM factuality risk is highest and the stakes highest for the youngest). T1 flow: LLM drafts → automated gate → human writes/rewrites to 10/10 → locked into a curated T1 bank. If a Friday theme has no signed-off T1 variant, the child at T1 gets the most recent evergreen curated T1 deck rather than an unreviewed one — the drop never blocks, but an unchecked deck never reaches a 4-year-old. T2–T4 may ship on ≥9 auto+human; T1 is the only tier gated to a curated allowlist.

4. Voice-mode adaptation

A quiz plays over voice inside the existing mode:"quiz" conversation. LoLo reads the question aloud (v1_synthesizeSpeech, Zephyr), the child answers by speaking, STT transcribes, the client grades locally against accepted/choices.

  • STT keyword biasing per question (#165). Before opening the mic for a question, send that question's expected answer terms — choices* + accepted.keywords* + kidVariants* — as speech-context phrase hints to the chirp_3 streaming server (elumi-kids-api/server :3005). This sharply lifts recognition of the domain word ("gills", "Kiemen") the child is most likely to say, which is otherwise easily misheard. Hints are per-question and swapped each turn.
  • Grading spoken answers. Normalize the transcript (lowercase, strip filler "um/ähm", singular/plural fold) then match against keywords ∪ kidVariants. Binary/mcq3: match spoken choice text or its ordinal ("the first one"). Full keyword hit → correct; concept-without-keyword → partial (open only); no match → incorrect.
  • Repair prompts (unclear answer). If STT returns low confidence or an off-topic transcript, LoLo does one warm repair, not a hard re-ask: "I didn't quite catch that — was it the first one or the second?" (binary/mcq3) / "Tell me again in your own words?" (open). Max one repair per question; a second miss reveals the answer gently and moves on — never trap the child.
  • Dead-mic pattern reuse (#179). Reuse the shipped dead-mic guard: if the mic never opens or returns silence within the window, LoLo prompts once ("Your turn — tap the disc and tell me!"), keeps the recorder armed (per reference_open_chat_3turn_audio_pattern — don't stop() the recorder between turns), and falls back to on-screen tap choices so a broken mic never stalls the quiz.
  • No timer. Voice mode especially must not pressure — the child may think out loud; LoLo waits.
