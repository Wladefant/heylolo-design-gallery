Appendix D — Quiz-deck authoring spec essentials (deck anatomy + question shapes) → feature 7

📖 Full documents: https://heylolodesign.wladefant.de/refs/quiz-deck-authoring-spec.html · the Sep–Dec deck run https://heylolodesign.wladefant.de/refs/quiz-decks-sep-dec.html

Deck anatomy: one deck = one theme × four tier variants × 5 questions each (founder spec: 5 questions per session). A deck maps to a Wonder Shelf slot on completion (mints the theme's interest/seasonal slot + the "Quiz Ribbon"). Decks are pre-authored and served to all (~$0 to run).

The four question shapes:

  • binary — Yes/No (the only type allowed at T1 4–5; concrete, observable, one fact).
  • mcq3 — 3 choices, exactly one correct (introduced at T2; also used T3/T4).
  • open-with-keywords — a short spoken/typed answer graded against accepted keywords + real "kid phrasings" (T3+ only).
  • open + why — open plus a "why do you think" follow-up (T4).

Per-question content: the question (EN+DE, German written age-native, not translated); the LoLo reveal lines — correct (celebrate + name the key idea, bold the big word), incorrect (never "wrong" — "Almost! / let's look" then the fact), partial (open only: child had the idea but missed the word → supply it warmly); and a funFact shown after the reveal (the payoff — never before, it would leak the answer).

Tier writing rules: T1 binary-only and hand-verified/curated (LLM factuality risk is highest and the stakes highest for the youngest); one "big word" max per deck at T2, bolded and defined in the same breath; never negate a stem for T1–T2 ("which does NOT…" confuses early readers). In-chat UI is built — your job is auditing the shipped flow, not authoring decks.

📖 Full document: https://heylolodesign.wladefant.de/refs/calm-corner-scripts.html

Each ritual runs 60–120s, has a fixed warm entry line and a fixed exit line (never abrupt), and the haptics engine paces the breath (a continuous inhale/exhale texture + a completion tap).

Ritual A — Belly-Breathing Balloon (4-4 pace, T1-friendly, ~70s):

Appendix J — Restrict-topics AVOID / CARE / SAFETY model → features 5, 44 (safety-adjacent context)

📖 Full document: https://heylolodesign.wladefant.de/refs/restrict-topics-deflection-spec.html

Three response modes decide how LoLo behaves when a sensitive topic surfaces, evaluated top-down (first match wins). We never build a wall — a blunt "we don't talk about that" shames the child and spikes curiosity. We acknowledge, then warmly pivot; and we never deflect a child's real feelings, only their curiosity.

  1. SAFETY-OVERRIDE — danger signals to the child's wellbeing (self-harm, abuse / "not safe at home," someone hurting them, running away, extreme fear). LoLo never deflects: warmth + validate + actively bridge to a trusted grown-up NOW; an immediate parent alert fires (not the digest). This always wins, over any topic setting, over any persona, mid-ritual, instantly.
  2. CARE — the child is personally affected by a heavy topic (grief, a real event) OR the parent tagged it a care-topic. LoLo validates the feeling → one age-true minimal sentence → warm bridge to Mama/Papa → keep-the-door-open. A digest note for the parent.
  3. AVOID — the child asks about a restricted topic out of curiosity (no distress). LoLo acknowledges → warm pivot to an adjacent allowed interest → no shame, no "forbidden" framing, and stays calm/consistent on repeat probing (varies the pivot, never "I already told you"). Silent log only.

The load-bearing distinction is curiosity vs. personal experience. "What is war?" → AVOID (deflect). "My grandpa died in a war and I'm sad" (same topic) → CARE (you never deflect real grief). Restriction gates information, never the child's own emotions. Never say "I can't talk about that," "That's not allowed," or any disclaimer that interrupts the flow (the robotic-refusal failure mode). This model is why Calm Corner (feature 44) must interrupt instantly on a SAFETY fire, and it colours the age-tier deflection tone (feature 5): T1 pivots to imagination/play; T4 respects autonomy and offers a real adjacent topic.
