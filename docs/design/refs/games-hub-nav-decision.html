Games Hub navigation — decision memo

status: RECOMMENDED (founder can veto) · 2026-07-05 context: #149 build starts ~W11 (Sep 14); BUILD-GUIDES flagged this as "a nav decision, not code". Today (child)/(tabs)/_layout.tsx has 3 native tabs.

Options

A. Push-from-Home (recommended for v1.2). A "Games" tile/entry on the child home pushes the hub as a screen. No tab-bar change, no native re-layout, zero App Review surface, ships in the v1.2 update without touching the shell. Cost: one extra tap; discovery depends on the home tile being visible.

B. Fourth native tab. Best discovery, but: crowds a 3-tab kid UI (each tab loses ~25% width), makes the hub feel core before it has content, and every later experiment (Learning Studio #150 — a 5th tab?) compounds the problem. Tab real estate is the scarcest UI resource in a kids app.

C. Hub replaces a Home section (games rail on Home). No nav change at all — the hub IS a horizontally-scrolled rail of game tiles on the child home, with "see all" pushing the full grid. Cheapest, most discoverable for the youngest tier (no navigation concept needed at 4–5).

Recommendation

C for v1.2, A's full-grid screen behind its "see all", revisit B only when ≥6 tiles are live AND usage data shows the rail saturating. Rationale: (1) the 4–5 tier can't navigate tabs reliably — a rail on the surface they already live on wins the youngest users; (2) launch tiles are few (chat-games + Connect Four + Memory + quizzes) — a full tab would look empty, a rail looks curated; (3) zero freeze/App-Review interaction; (4) Learning Studio (#150) later follows the same rail pattern, avoiding tab inflation.

Age-tier note (#105): the rail's tile set is tier-filtered — T1 sees 3 big tiles, T4 sees the full set with difficulty badges.

Decision needed from founder by ~Sep 8 (v1.2 build start). Default = C if no veto.
