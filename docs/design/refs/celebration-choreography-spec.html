Celebration Choreography Spec — HeyLolo

The definitive micro-spec of every reward moment in HeyLolo. Every celebrate moment made buildable: haptic + sound + motion + LoLo voice line, per tier, per trigger. Unifies the gamified-UX mechanics, wonder-shelf catalog, mascot motion state machine, voice-conversation design, and the real feedback API.

Status: COMPLETE · 2026-07-06. For GitHub #54 (feedback module) + #128 (mascot) + #142/#146/#158/#176 (celebrate-able events). Builds on (does not repeat): gamified-ux-winning-mechanics.md §3 (the R4 arousal arc; "no big-bang confetti on routine events"), wonder-shelf-launch-catalog.md §2 (earn-rule → tier table), mascot-motion-state-machine-spec.md §1.1 (states S6–S10), kids-voice-conversation-design.md §2.2 (the Gasp / Whisper-Secret devices), calm-corner-scripts.md (the [haptic: verb] notation style), and the SHIPPED feedback API in src/shared/feedback/.

The one-line thesis (research verdict): the rare-celebration choreography is where premium kids apps win. Juice must "hit the eyes, the ears, and the hands simultaneously" — but overused it becomes a crutch that gets the feature turned off (gamedesign.gg; wayline.io). Apple says the same for haptics: "use them sparingly… overuse gets the feature turned off" (Playing haptics, HIG). So HeyLolo's celebration system is a rarity budget: magnitude scales with meaning, the biggest beats fire ~once in a child's lifetime, silence is a deliberate tier.

Variable-MAGNITUDE, not variable-PROBABILITY (the deterministic-tier rule): variable-ratio schedules — reward after an unpredictable number of responses — are the compulsion engine, and the loot-box↔gambling link is stronger in minors (supportivecareaba; gamified §2). HeyLolo never varies whether or how likely a celebration is, only how big — a deterministic function of the trigger. Surprise lives only in content (which wonder), never in the schedule. That is the whole ethical difference from a slot machine.

1. The tier system (unified)

Four tiers, escalating. The real shipped API is feedback.success("S" | "M" | "L") + feedback.error() + discrete haptic verbs (hSelection, hImpactLight/Medium, hNotifySuccess) — there is no celebrate() function yet (the other docs use celebrate(tier) as shorthand). The mapping below is the canonical translation, and §5 proposes the thin celebrate(tier) wrapper that unifies haptic + sound + mascot + voice into one call (today only the haptic+sound half exists).

TICK WHISPER MOMENT LANDMARK
What it marks routine effort — a turn taken, a face tapped, a slot silently filled a small named win — one artifact minted, a nice exchange, first-of-something a gated real win — quiz deck done, game won vs LoLo, story finished a rare peak — a set completed, an age-up (#158), a certificate (#176)

Haptic (real verb) hSelection or hImpactLight — discrete, no success() feedback.success("S") → SUCCESS_S (rising 2-bump) feedback.success("M") → SUCCESS_M (3 rising taps + swell) feedback.success("L") → SUCCESS_L (4-tap crescendo, near-peak amp)
Sound (earcon) silent (haptic-only — feel policy) silent-to-soft — one warm marimba note, OR silent (see §5 gap) the shipped 3-note pentatonic arpeggio C5→E5→G5 (feedback.success sound) a distinct bigger earcon — arpeggio + a bell() shimmer top + longer tail (NEW, §5)

Frequency cap uncapped soft: no more than a few/session hard: never two MOMENT+ back-to-back (pacing guard); ~1–3/session very hard: ≤1/session, often ≤1/week; gated by real milestones only

What NEVER moves, at any tier (the juice-restraint invariant): the composer/input bar, the nav/tab bar, the child's own prior messages, and the parent-facing chrome. Celebration motion is local to the mascot + the earned artifact. No screen-shake (that's an impact metaphor, wrong for a warm companion), no modal that blocks the next turn, no full-screen confetti except the single soft glow-wash reserved for LANDMARK. Reduce-motion collapses every mascot REACT to its opacity/glow fallback (mascot spec §1.1); the haptic + (rare) earcon still carry the beat.

Silence as choreography. TICK is silent-and-still by design; WHISPER is usually voice-silent. This is not underdelivery — it is the rarity budget that makes MOMENT feel like a win and LANDMARK feel like a landmark. A child who gets a spoken "you did it!" on every turn stops hearing it. LoLo's voice at a celebration is itself a scarce resource.

2. Trigger → tier registry (canonical)

Every celebrate-able event in the product, its tier, and any special variant. This is the single source of truth; wonder-shelf-launch-catalog.md §2 maps the artifact per event, this maps the choreography. Issue numbers on Wladefant/heylolo-app.

A single Wonder-Shelf artifact minted (#142) WHISPER artifact flies to shelf; Day-0 first-ever artifact bumps to MOMENT (the guaranteed 3-min payoff, #179)
First utterance / first message ever WHISPER bump to MOMENT on Day-0 (mints Hello Spark)
First mood check-in, first question WHISPER one artifact each
Tomorrow's-Seed payoff on next open (#157) WHISPER LoLo may speak (it's a promise kept) — the one WHISPER that often gets a line

Streak day / login count NEVER no streaks exist (anti-list) — see §3

Tier-bump rules (deterministic, not random): (1) the first-ever instance of a WHISPER event on Day-0 bumps one tier to MOMENT (the onboarding delight guarantee, #179); (2) a set-completing artifact fires WHISPER for the mint then LANDMARK for the completion — but as ONE choreographed sequence, not two celebrations (the pacing guard collapses them: the artifact flies in, then the capstone blooms). No other bumps. Everything else is a fixed lookup.

3. The anti-spec — what NEVER gets celebrated, and why

Restraint is the product. Each of these is a place a lesser app fires juice and HeyLolo deliberately stays flat. A celebration here would spend the rarity budget on a non-event and train the child to expect fanfare for nothing — the exact overuse Apple and the game-feel literature warn kills the feature (Playing haptics, HIG; wayline.io).

Event Why it gets NOTHING (not even a TICK, unless noted)
Opening the app / cold start LoLo waking is the greeting (mascot S1 WAKE) — that's character, not reward. Celebrating app-open trains attendance-for-reward, the login-streak psychology we reject (gamified §2).
Screen / route transitions Navigation is not an accomplishment. A hSelection tick on a button press is fine (that's touch feedback), but arriving on a screen earns nothing.
Purchases / subscribe / paywall The hardest rule. A celebration on purchase turns the child into the sales channel — the exact Prodigy/FTC failure mode (gamified §4.1). The child UI never celebrates money. The parent may see a plain confirmation; the child sees nothing.
Parent actions (settings changed, digest opened, child added) The parent surface uses plain system feedback, never the child's warm celebration vocabulary. Different audience, different register — celebration belongs to the child's earned moments only.
A correct answer mid-quiz Only the completed deck is a MOMENT. Per-question fanfare turns a quiz into a slot machine of micro-rewards and punishes the wrong answers by contrast. Mid-quiz gets, at most, LoLo's warmth in voice (a Gasp on a cool fact), never a success().
A wrong answer / a game loss / a mishear No error earcon, no sad haptic. feedback.error() is reserved for genuine technical failure only (no network, recorder died) — never for a child being wrong. A mishear routes to mascot S12 error-kind (a gentle "hmm?"), which is repair, not punishment.
Streak maintained / daily return No streaks exist. Returning is met with delight-in-content ("I found a new wonder"), never a counter that could break.
Time-limit reached / session end The wind-down is a calm register (mascot S11), the opposite of celebration — ending calmly is the parent-side retention win, and a fanfare at the lock screen would spike arousal at exactly the wrong moment.

4. Three fully-choreographed examples

Timeline notation (calm-corner style): T+Nms · [haptic: verb] · [sound: earcon] · [mascot: state] · [motion: what moves] · LoLo: "…". T=0 is the instant the earning event resolves. All three obey: build → peak (≤2–3 s) → settle; nothing but the mascot + artifact moves; reduce-motion swaps mascot REACT for opacity/glow.

4a. WHISPER — a wonder is minted

Trigger: the child asks their 3rd question and it mints the Curious Compass artifact.

T+0     [mascot: S6 gasp]           LoLo does a quick eyes-wide inhale — the Gasp precedes the reward
T+120   [haptic: success("S")]      the rising 2-bump (SUCCESS_S) — felt, not heard
T+150   [mascot: S8 celebrating-small]  a single bob
T+180   [motion: artifact spawns at mascot, arcs to the shelf corner over ~350ms]
T+200   [sound: one soft marimba note ~C5]   OR silent if session sound-budget spent (§5)
T+220   [motion: ONE sparkle pops where the artifact lands — nothing else moves]
T+550   [mascot: settle → back to S3 listening]
        LoLo (usually SILENT here; if she speaks, the Whisper-Secret register):
        (soft) "…ooh, that's one for our shelf."

One-liner: a gasp, a 2-bump, a bob, and the compass flies to the shelf — she barely says a word.

4b. MOMENT — a quiz deck is completed

Trigger: the child answers the last card of a 6-card dino deck (#121); mints the Quiz Ribbon.

T+0     [mascot: S4→hold]           a tiny "here it comes" beat (anticipation — the build)
T+90    [haptic: success("M")]      3 rising taps over a swelling bed (SUCCESS_M), sharpness 0.4→0.6
T+120   [sound: success arpeggio]   C5→E5→G5, peaks building, sparkle on the top note (the shipped earcon)
T+140   [mascot: S9 celebrating-medium]  a hop + small sparkle burst
T+160   [motion: Quiz Ribbon arcs to shelf; a small local burst blooms around the mascot]
T+600   LoLo (process praise, ONE line — NOT a score):
        "You finished the WHOLE dino deck! You kept going all the way to the end."
T+1200  [mascot: settle → S5 speaking (for the line) → S3]   full beat done under 1.2s of motion

Variant — quiz perfect: identical tier/haptic/sound; only LoLo's line changes ("…and every single one — whoa!"). Deliberately NOT bumped to LANDMARK. One-liner: anticipation beat → 3-tap swell + the bright arpeggio → a hop → "you finished the WHOLE deck" — a real win, once per deck.

4c. LANDMARK — an age-up (#158)

Trigger: a competence signal + birthday crosses the child from T1 into T2; the rarest beat, ≤1/session and often ≤once in months.

T+0     [mascot: S4 thinking → hold]   LoLo pauses, "notices" — the longest build (this is allowed to breathe)
T+250   [haptic: success("L")]         the 4-tap crescendo (SUCCESS_L), brightest peak amp ~0.85 — allowed
                                        near-peak ONLY because it fires ~once in a child's lifetime (rarity = safety)
T+300   [sound: LANDMARK earcon]        the arpeggio + a bell() shimmer top note + a longer warm tail (NEW, §5)
T+350   [mascot: S10 celebrating-large]  full jump + arms up + Skia particle bloom
T+400   [motion: soft full-screen GLOW-WASH (not confetti); 1–2 new capability tiles fade in on the hub]
T+700   LoLo (she notices growth OUT LOUD — the age-up line):
        "Hey… you're getting bigger! I think you're ready for the big-kid questions now.
         Look — I brought you some new things to try."
T+2000  [mascot: settle → S2 idle]      the glow-wash dissolves; total ≤2.5s
        [parent digest gets a "LoLo is growing with them" line — the renewal signal, gamified §5]

One-liner: LoLo pauses, notices you grew, the 4-tap crescendo + bell shimmer bloom, and new tiles quietly appear — the rarest beat, earned, never repeated.

5. Implementation notes — what exists vs what needs adding

What EXISTS in src/shared/feedback/ (the real API)

  • feedback.success("S"|"M"|"L") → playSuccessPattern() plays the shaped Pulsar haptics SUCCESS_S/M/L (haptic-patterns.ts), degrading to hNotifySuccess when the native module is absent/incapable. This is the whole haptic half of WHISPER/MOMENT/LANDMARK — already built.
  • The reward earcon: success() plays the C5→E5→G5 marimba/voice arpeggio on the shared audio context (index.ts:135, synth.ts marimba/voice/bell).
  • Discrete verbs for TICK: hSelection, hImpactLight/Medium (haptics.ts).
  • playLandmarkPattern() exists (haptic-patterns.ts:130) but is wired to nothing — "no such event exists in the app yet."
  • Graceful degradation, the global Vibrations toggle, and the parent gamification toggle are all already respected.

GAPS — what this spec requires that is NOT built

  1. No celebrate(tier) wrapper. Callers today must call feedback.success(mag) for the haptic+sound, then separately drive the mascot and any voice line. This spec's whole point is that the four channels are ONE choreography. Add celebrate(tier, opts) that: fires the haptic magnitude, fires the tier's earcon, sets the mascot celebrate+celebrateTier inputs (mascot spec §1.3), and emits the voice-line policy. One call, four channels.
  2. Sound does not scale with tier. success() plays the same arpeggio for S/M/L. Per §1 the tiers need distinct earcons: WHISPER = one soft marimba note (or silent under budget); MOMENT = the current arpeggio; LANDMARK = a NEW bigger earcon — arpeggio + a bell() shimmer top + a longer tail. bell() already exists in synth.ts; this is a small additive recipe, not new infra.
  3. TICK must NOT call success(). It's a discrete hSelection/hImpactLight only — silent, no mascot celebrate state. Make sure the registry routes TICK away from the success path.
  4. No pacing guard. "Never two MOMENT+ back-to-back" is unenforced. Add a module-level timestamp latch in celebrate(): a MOMENT/LANDMARK within ~N s of a prior one downgrades to a silent artifact-only mint or defers the fanfare. Deterministic.
  5. Mascot celebrate states S8/S9/S10 are Rive phase-2 (mascot spec §2), not shipped. Until then celebration motion is the Skia sparkle/bloom + artifact arc only; wire the mascot inputs now so they light up when Rive lands.
  6. LoLo voice line on celebrate has no mechanism. Voice is BE-persona-driven. v1: a small client line-bank keyed by tier+trigger, in LoLo's Gasp/Whisper registers, TTS'd via the existing path; silence is default for TICK/WHISPER. Later: a BE [celebrate:moment] marker the persona fills.
  7. Session sound-budget. Cap audible earcons per session (WHISPER goes silent after the first 1–2); haptic always fires. A counter on the earcon context.

The config shape

A single celebration-registry.ts (versioned code config, same discipline as wonder-collections.config.ts):

type Tier = "tick" | "whisper" | "moment" | "landmark";
// TIER_SPEC = the §1 table verbatim, keyed by Tier:
//   { haptic: "selection"|"S"|"M"|"L", earcon: "none"|"soft"|"arpeggio"|"landmark",
//     mascot: null|"S8"|"S9"|"S10", voice: "never"|"optional"|"short"|"beat",
//     maxMotion: "target"|"artifact"|"burst"|"glowwash",
//     cap: { perSession?, cooldownMs? } }
// TRIGGERS = the §2 registry, keyed by event → { tier, variant?, bumpDay0? }.
//   Anti-spec events are simply ABSENT — no entry = no celebration.

celebrate("set.complete", { childId, artifactKey });

The anti-spec (§3) is enforced by absence: no registry entry → nothing. The pacing guard and Day-0 bump live in celebrate(), deterministically.
