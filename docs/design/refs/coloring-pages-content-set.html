0. TL;DR

24 launch pages: 12 evergreen (3 × 4 interest collections, matching Wonder Shelf's Dino Dig / Star Voyager / Animal Friends / Ocean Deep) + 8 seasonal (autumn, Halloween, St. Martin, winter/Christmas — timed to the content-drop-economy.md §7 calendar) + 4 LoLo-themed (descriptor-locked to LoLo, Pip the ghost, Anneli the owl). Pure black-and-white line art via Imagen 4 Fast, tiered by line density (T1 simple → T3 detailed), QC'd for closed regions, printable at A4.

1. Line-art generation spec

Engine: reuse the shipped Imagen 4 Fast pipeline (imagen-4.0-fast-generate-001, europe-central2, ~2.7s/image, $0.039/image — ai-chat.service.ts:165). Coloring pages are pure line art, so the prompt swaps the wonder-shelf/story-pack color-lock suffix for a monochrome-lock suffix — same discipline, opposite palette instruction.

Prompt template (subject + tier clause + fixed style-lock suffix):

"A single {SUBJECT_DESCRIPTION}, {TIER_LINE_CLAUSE}, pure black-and-white line-art coloring-book illustration, bold clean unbroken outlines, every region fully enclosed for flood-fill, ZERO shading, ZERO gray fills, ZERO gradients, ZERO cross-hatching, ZERO background texture, plain white background, NO text, NO letters, NO watermark, friendly rounded storybook shapes matching HeyLolo's house style, centered composition, printable at A4."

Tier line-density clause ({TIER_LINE_CLAUSE}) — mirrors the F2 age tiers (T1 4–5 / T2 6–7 / T3 8–9 / T4 10–12):

  • T1 (simple): "5–8 large simple regions, extra-thick outlines (~6–8pt at print scale), one clear subject, no background clutter, smallest region no smaller than a thumbprint"
  • T2: "8–14 medium regions, thick outlines (~4–6pt), one subject + light background detail"
  • T3 (detailed): "15–25 varied regions, medium outlines (~3–4pt), one subject + a fuller scene background, fine detail allowed inside large shapes only"
  • (T4 reuses T3's density — coloring pages top out at T3 detail; older kids get freehand brush + finer palettes, not denser line art.)

QC criteria (Brandy pass, same discipline as the Wonder Shelf art QC):

  1. Closed regions only — every outline forms a fully bounded loop; a flood-fill tester must not "leak" into a neighboring region or off the page. This is the #1 reject reason.
  2. No gray fills / no anti-aliasing halos — lines must be pure black on pure white (high-contrast threshold), not soft-edged gray, or the in-app bucket-fill tool mis-detects edges.
  3. Minimum region size — no enclosed region smaller than ~6mm at printed A4 scale (T1) / ~4mm (T2–T3) — small-finger tap-to-fill fails on regions that small.
  4. Single coherent subject, no embedded text/letters, no scary/ambiguous elements (brand rule, same as seasonal-friendly constraints in the Wonder Shelf catalog).
  5. Style consistency across the 24-page set (same line weight family, same "friendly rounded" house shape language) — reject/regenerate outliers.

Generate 2–3× overage per page (Imagen is cheap), curate the best pass — identical production discipline to the Wonder Shelf's 88-artifact batch (§4 of that catalog).

Print variant (A4): portrait A4 (210×297mm), 300 DPI export for print, pure 1-bit-style black line on white (no color layer). Safe margins: 15mm left (binder/hole-punch allowance), 10mm top/right/bottom. Reserve a ~20mm footer strip with a small LoLo logo mark + a hand-write line "Name: _______ / Für: _______" (EN/DE). Export as PDF (print) alongside the in-app PNG/vector-ish raster (flood-fill target).

4. Conversation-to-coloring hook (#187)

Trigger rule: if a child spends ≥5 cumulative minutes exploring one topic in a single conversation (chat or Explorer), LoLo makes one soft, in-chat offer — never a push notification, never repeated if declined (matches the DSA-safe "no engagement push to the child" discipline already applied to drops, content-drop-economy.md §1/§4): "Want a [topic] page to color? I could make you one!" / "Möchtest du ein [Thema]-Bild zum Ausmalen? Ich kann dir eins machen!"

Resolution order on "yes":

  1. Match first — if the topic maps to one of the 24 pre-vetted launch pages (§2) or a future seasonal/collection page, offer that page instantly (zero generation cost, guaranteed QC-passed).
  2. Generate fresh only if no match exists — build {SUBJECT_DESCRIPTION} from the conversation topic, run it through the §1 prompt template at the child's age tier (F2), and generate via Imagen 4 Fast.

Generation guardrail (freshly-generated pages only): any page not from the pre-vetted launch set must pass, in order:

  1. The existing image content-safety moderation pass (the same classifier gating chat-generated images, per the F30 safety pipeline) — reject anything scary, ambiguous, or off-brand.
  2. The §1 region-QC heuristic (closed-region check, no gray-fill detection) — reject anything the flood-fill tool can't parse.
  3. On any failure, fail silently to the child — no error shown — and fall back to offering the closest-matching pre-vetted evergreen page from §2 instead. The child never sees a rejected or broken image.

This is the off-app bridge concept from the feature-mining research (N25, "Conversation-to-Printable"): it converts screen time into an off-screen, fridge-worthy artifact, using the same export/print path as §1, and is deliberately distinct from F24's in-app coloring of fixed/pack line-art — #187 is generate-from-conversation, #127 is color what already exists.
