1. The 12 launch collections

Every artifact is a small collectible token (a fossil, a shell, a candle) minted by ONE deterministic event. Sets are 6–10 items, complete-able, no numbers shown to the child. Each set completing fires the set-complete arc ONCE (celebrate("landmark")); each single artifact drop fires a small tick (celebrate("whisper")). Art descriptions are the Imagen/Brandy brief (§4 style-lock applies to all).

2. Earn-rule spec table

Celebration tiers map to the shipped celebrate(tier) feedback module (src/shared/feedback/): tick = whisper (haptic + micro-sparkle, per single artifact) · set-complete arc = landmark (the R4 build→≤2–3 s peak→settle, once per set). A first-of-day artifact may bump to moment.

# Event (real, shipped/planned) Artifact granted Collection Tier
1 First voice conv / first message Hello Spark First Adventures tick (moment on Day-0)
2 First mood check-in Mood Pebble First Adventures tick
3 First question asked Curious Compass First Adventures tick
4 Tomorrow's Seed payoff (#157) Wonder Seed First Adventures tick
5 Explorer discovery (#140) Explorer's Lens → +interest set slot First Adventures / interest tick
6 Calm Corner ritual (#147) Calm Shell / Calm Lavender First Adv. / Feelings Garden tick
7 Quiz deck completed (#121) Quiz Ribbon → +interest deck slot Little Thinkers / interest moment (a gated win)
8 Game won vs LoLo (#107/#108) Duo Trophy → +interest game slot LoLo's Friends / interest moment
9 Story finished (#126) Storyteller's Feather / Idea Lightbulb LoLo's Friends / Little Thinkers moment
10 Friday drop opened (#152) Friday Gift Box + seasonal slot-1 LoLo's Friends / seasonal tick
11 Bedtime session (#164) Bedtime Moon / Cozy Nightlight LoLo's Friends / Feelings Garden tick
12 Age-up (#158) (unlocks next-tier interest set + Age-Up landmark) interest landmark
13 Certificate earned (#176, later) Scholar Scroll Little Thinkers moment
14 7/7 slots filled in any set Golden capstone (Egg/Star/Acorn/Anchor/…) that set set-complete arc (landmark)

Day-0, first 3 minutes (#179): onboarding → LoLo wakes → first spoken/typed exchange (~60–90 s) mints Hello Spark with a whisper tick and the artifact visibly flies to the shelf → the mood check-in (~2 min) mints Mood Pebble. Two artifacts before minute 3, no numbers, no gate.

Something every session (deterministic, never random): early sessions always hit a First-Adventures / LoLo's-Friends first-time or tenure rule (5-turn, 5-session, morning-window). Steady-state, a kid asking questions / doing a quiz / an Explorer capture always mints an interest or seasonal slot. The guarantee holds as long as one incomplete set exists — and the seasonal cadence (a fresh drop set opening every few weeks, #152) ensures a child is never fully done, so there is always a slot to fill. No fallback RNG is used; if literally every active set is complete, the next seasonal drop is what re-opens earning (by design, not chance).

3. Anti-dark-pattern checklist (each rule shown passing)

Rule (from research anti-list) How the catalog passes
No loss / no decay Artifacts are permanent; sets never expire; a missed Friday/seasonal set re-runs annually (Toca Gift-Bonanza pattern). Nothing is ever removed or reset.
No numbers / XP / streaks Zero point counts, zero "12/50" bars shown to the child, zero streak calendar. Progress is visual (filled vs empty slot silhouettes on the shelf). Internal progress counters exist BE-side only, never rendered.
Celebrate once Single artifact = whisper tick; set completion = ONE landmark arc. No back-to-back confetti; pacing guard prevents two moment+ celebrations in a row.
Deterministic earn Every row in §2 is a fixed event→artifact rule. No variable-ratio, no mystery box, no wheel. (DSA: random rewards off-by-default for minors — we simply don't have any.)
No child-visible paywall pressure See gating below — the child UI shows no padlocks that flaunt a purchase, no "upgrade to unlock", no two-class cosmetics (the Prodigy FTC failure mode).
No social graph No "friends" artifacts, no stranger comparison, no shared/global shelf. Family view (siblings) is parent-opt-in only.
Parent toggle The whole shelf + all celebrations sit behind the existing gamification parent switch (BUILD-GUIDES §2d), conservative default; off = surface hidden, engine still records silently.
No engagement push to child Award engine emits nothing to the child device as a notification; "earned while away" lands only in the parent digest (#135).

Plus-gating recommendation — NONE of the 12 launch sets are child-visibly gated. All 12 are earnable by any child on any plan at launch; this is the deliberate anti-Prodigy stance and the premium brand's differentiator. Monetization of the shelf, if ever, is additive and invisible: post-launch extra seasonal/interest sets can be Plus-only, but a non-Plus child simply never sees that set exist (no teasing padlock, no "ask a grown-up to unlock"). Rationale: the shelf's job is retention + the parent-visible progress that drives the 2–3× renewal multiplier; turning it into a child-facing upsell would trip Apple Kids 1.3 and hand Fairplay the exact press angle that sank Prodigy. Sell the breadth of sets to the parent on the paywall, never to the child on the shelf.

4. Art production plan (~102 artifacts)

Engine: reuse the shipped Imagen 4 Fast pipeline (ai-chat.service.ts:165, imagen-4.0-fast-generate-001, europe-central2, ~2.7 s/image, $0.039/image). A throwaway batch script calls generateImage(prompt,"illustration") over the artifact manifest — not a new callable, a one-off tool run.

Style-lock prompt template (identical suffix on every artifact, only the object noun changes — same trick the Story pipeline uses for character consistency, §1f of BUILD-GUIDES):

"A single {ARTIFACT_OBJECT}, centered, one collectible token, soft rounded chunky shapes, thick clean outlines, matte storybook illustration, warm tide-pool palette (teal, coral, sand, cream), soft drop shadow, gentle inner glow, friendly and cozy, NO text, NO letters, NO background clutter, plain soft-cream background, consistent flat children's-app icon style."

Per-artifact {ARTIFACT_OBJECT} = the 1-line brief in §1 (e.g. "a T-Rex tooth fossil in warm amber tones"). Seasonal/friendly constraints baked in ("friendly cartoon ghost, not scary" for Halloween).

Brandy QC pass: generate ~2–3× overage per slot (Imagen is cheap), Brandy curates the best one per artifact for (a) style/palette consistency across the 88, (b) kid-safety (nothing scary/ambiguous), (c) single-object clarity at 96 px shelf size. Reject/regenerate the ~20–30 % that drift. The golden capstones get a hand-check for the extra glow treatment.

Effort estimate: manifest + batch script ~2 h · generation runs ~1 h wall (102 × 2–3 passes, unattended) · Brandy curation+regeneration ~3–4 h · export/rename/pack ~1 h. ≈ 6–8 hours total (one designer-day with Claude). API cost with 3× overgen ≈ ≈$11.93 (306 images × $0.039).

File/naming convention: wonder-shelf/<collection-key>/<artifact-key>.png, kebab-case, e.g. dino-dig/trex-tooth.png, first-adventures/hello-spark.png. 512×512 PNG, transparent or soft-cream background, @2x/@3x optional (single-object icons upscale cleanly, so one size is acceptable v1).

Where they live: bundle all 12 launch sets in-app under src/shared/assets/wonder-shelf/ — mirrors how badges-baseline.html already pulls assets/mood/*.png / assets/ds/*.png; 88 small PNGs (~a few MB) is fine to ship, gives zero-latency offline render and versioning. Post-launch drop sets (new seasonal/interest sets delivered via the Friday-drop system without an App Store update) are served from Firebase Storage and referenced by URL in the drop config — so the shelf can grow without a binary. Rule of thumb: known-at-launch → bundled; delivered-later → Storage.

5. Firestore additions to the BUILD-GUIDES shelf schema

BUILD-GUIDES §2c already defines earned-state on child-achievements with collections: Array<{key, itemRefs, completedAt}>. Two refinements needed for deterministic per-slot granting, plus one optional config collection for drop-delivered sets.

5a. Refine collections[] to name the filled slot (so each artifact-key is idempotently grantable):

collections: Array<{
  key: string                       // "dino-dig", "first-adventures", ...
  items: Array<{                    // was itemRefs: string[]
    artifactKey: string             // "trex-tooth" — the named slot
    sourceRef: string | null        // id of the message/quiz/story/explorer capture that earned it
    grantedAt: Timestamp
  }>
  completedAt: Timestamp | null     // set when items.length === definition.slotCount
}>

The award trigger (BUILD-GUIDES §2d, Firestore trigger — deterministic, idempotent) checks artifactKey presence before appending, so replays never double-grant.

5b. Artifact/collection DEFINITIONS stay as versioned code config by default — a wonder-collections.config.ts (shape below), same discipline as badge defs / plan rules in feature-access.rules.ts (tamper-proof, versioned, no BE round-trip to render):

{ key, titleEn, titleDe, theme, tier?, slots: Array<{
    artifactKey, nameEn, nameDe, art, earnRule   // earnRule = a typed event descriptor
  }>, capstoneKey?, season?, plusOnly?: boolean }

5c. Optional Firestore wonder-collections config collection — ONLY for drop-delivered sets. Post-launch seasonal/interest sets that ship via the Friday-drop scheduler (#152's weekly-drops) live as config docs mirroring the §5b shape, so a new set can be added without an app update. The FE merges code-config sets (launch 12) with any wonder-collections docs. Launch does not require this collection — the 12 sets ship as code config; add wonder-collections when the first post-launch drop set is authored. plusOnly here is the only gating flag, and it hides the set entirely from non-Plus children (§3), never renders a padlock.

No new fields beyond the above; no xp/points/streak anywhere (anti-list). Awarding remains a projection over existing data (messages, imageType:"generated", stories, quiz completions, Explorer captures, mood docs) — the events already exist; the shelf only maps them to artifact slots.
