1. The step grammar

A template is a fixed ordered list of primitive additions. Each step shows a soft "ghost guide" (a traced Skia path) the child draws over — no pass/fail; the overlay just advances when the child lifts a finger. Step 1 = the anchor (biggest shape, body or head). Middle steps add exactly one new part each — never two new elements in one T1/T2 step. Last step = the face or signature detail — it makes the character "come alive" and cues the reveal.

Difficulty ladder (reuses the #105 age tiers):

Tier Age Steps Shape budget
T1 4–5 3–4 One big/simple primitive per step. No paired parts, no detail steps.
T2 6–7 4–5 Primitives + ONE paired-shape step (two ears, two wings). Still no fine detail.
T3+ 8–12 6 Primitives + 1–2 detail steps after the big shape (spots, pattern, texture, expression).

The encouragement script — 3 slots, every step: (1) a delight beat naming the shape just added (Gasp/Sound-Effect device, kids-voice-conversation-design.md §2.2); (2) zero correction, ever — never "not quite," "stay in the lines," or any mention of wobble; (3) a forward nudge into the next step ("Now let's...").

"Every X looks different, that's what makes yours YOURS" fires ONCE per template, at the step where variation shows most (usually the face/pattern step) — never every step, per the ≤1–2-devices-per-turn repetition rule.

The reveal line follows the last step: a Gasp + naming the finished creature + an invitation toward the done-screen. Never comparative — always presenting-tense wonder.

2. 12 templates — 3 per Wonder Shelf interest collection

Each completed template mints that collection's existing "generate an image" shelf slot (Amber Bug / Comet Trail / Butterfly / Coral Branch — wonder-shelf-launch-catalog.md §1 sets 5–8) — a hand-drawn creation counts the same as an AI-generated one for that slot. This needs zero schema change: the award trigger already treats the slot as filled-once (idempotent), so if a child draws two templates in the same collection, only the first mints; the second is just fun. No new Firestore field, no new abstraction.

3. The share/finish moment

Done screen flow:

  1. The finished drawing renders full-screen (the child's own line art, captured from the Skia canvas).
  2. LoLo speaks the reveal line (§1) — voice + on-screen text; no thinking-sound needed (not live voice — a short reveal animation covers any gap).
  3. Auto-mint, no button. The shelf slot (§2) fills silently the instant the last step completes — the Wonder Shelf "no numbers, no gate" rule. The artifact flies to the shelf as part of the reveal beat, not a separate "claim" tap.
  4. Two calm affordances below the art: "Save this drawing" (camera roll) and "Draw another". No streak, no counter, no "drawing #4 today."

What's shareable vs. gated:

  • The drawing itself — de-identified line art, no name/face/PII — is freely shareable/saveable any time. This is the brief's "shareable de-identified artwork"; it needs no parent gate because it carries no PII (the constant bar the Brag Card precedent also uses).
  • A photo of the drawing-in-progress (e.g. a future print-and-trace mode where a child photographs a paper page) is different — it can capture a hand, a room, incidental PII. Per the Brag Card precedent (#163): default OFF, routed through the same parent-gated toggle used for milestone cards — never a new consent surface.
  • The share sheet reuses the existing Brag Card mechanism (#163): "{name} drew a {creature}!" caption + art, one-tap to OS share, parent-gated like any other milestone card.

4. Production notes

What Brandy draws per template: one full-color reveal illustration (Wonder Shelf style-lock — wonder-shelf-launch-catalog.md §4: tide-pool palette, thick outlines, no text) plus a step-overlay frame per step (3–6/template), delivered as clean vector SVG (not flattened PNG) so engineering converts them into Skia trace-guide Path data — a raster reference isn't tracing-usable.

Effort estimate: 66 assets total (54 step vectors + 12 reveals). Reveals reuse the Imagen 4 Fast pipeline (~2.7s/image, ~$0.039/image) — near-zero cost. Step vectors need hand authoring: ~40 min/template (T1) up to ~90 min (T3). Total: ~14–16 designer-hours, one Brandy sprint, + ~1–2 hours batch-export using the wonder-shelf naming convention (drawing-templates/<collection-key>/<template-key>/step-N.svg + reveal.png).

Seasonal cadence hook: the Friday-drop engine (#152) can deliver new templates with no App Store update — e.g. a Halloween pumpkin (T1, 4 steps: circle → 2 triangle eyes → jagged mouth → stem+glow) live in October. Per the wonder-shelf rule (known-at-launch → bundled; later → Storage): the 12 launch templates ship bundled; seasonal drops (pumpkin, snowflake, flower, heart) are Storage-served on the same drop config as seasonal Wonder Shelf sets — one mechanism, not two.
